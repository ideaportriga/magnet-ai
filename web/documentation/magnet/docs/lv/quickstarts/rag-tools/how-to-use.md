# RAG rīku izmantošana

## RAG rīku izmantošana aģentos

RAG rīkus var izmantot aģenti, kad aģents saprot, ka nepieciešams sniegt atbildi no zināšanu bāzes. Lai to izdarītu, konfigurējiet RAG rīku un pievienojiet to kā aģenta darbību.

[Lasīt vairāk](../../../lv/admin/configure/agents/other-configuration.md) par aģentu konfigurēšanu.

## RAG rīku izmantošana atsevišķi

Tipisks RAG rīku lietošanas gadījums ir J&A funkcionalitāte. Lai gan J&A var būt daļa no aģenta ar plašākām iespējām, to var izmantot arī atsevišķi un nodot gala lietotājiem kā daļu no [AI lietotnes](../ai-apps/overview.md) vai izsaukt tieši caur API.

RAG rīka publicēšana caur AI lietotni prasa tikai dažus klikšķus, lai pievienotu to konfigurējamam UI panelim, ko var iegult jūsu sistēmā ar minimālu iestatīšanu. AI lietotnes atbalsta Salesforce un Oracle Redwood tēmas un var saturēt vairākus AI rīkus, kas organizēti cilnēs.

Kad jūsu RAG rīks ir gatavs, vienkārši pievienojiet to AI lietotnei, iegūstiet ģenerēto URL un ieguliet lietotni savā CRM. Lai izmantotu vēl pielāgotāk, izsauciet AI lietotni pēc tās sistēmas nosaukuma.
